Преобразование типов

  •

Существуют языки программирования со статической и динамической типизацией. Давайте разберёмся, что это значит.

Статическая типизация определяется тем, что во время объявления переменной ей сразу задаётся тип данных. В такую переменную можно записать только тот тип данных, с которым она была создана.

// Java – статическая типизация

private boolean isTrue = true;

isTrue = "Some string"; // Ошибка! Тип значения не соответствует типу переменной.

Динамическая типизация. Все типы выясняются уже во время выполнения программы. То есть можно объявить переменную строкового типа, а затем записать в неё число. Интерпретатор на каждом этапе выполнения будет считывать новое значение и определять его тип.

JavaScript — язык программирования с динамической типизацией, потому что в нём все типы определяются интерпретатором на этапе выполнения программы.

// JavaScript - динамическая типизация

let isTrue = true;

isTrue = "Some string" // Ошибки нет. Тип переменной поменялся с булевого на строку

Примеры динамической типизации

Про типизацию в JavaScript ходит много смешных историй. Давайте разберём, насколько они правдивы.

Строки и числа

У нас есть задача от технического руководителя:

«Нужно сделать на сайте калькулятор, который прибавляет к значению ввода цифру 2».

«Пффф! Простейшая задача» — думаем мы и приступаем к реализации.

Первое, что мы делаем — получаем данные с сайта и записываем их в переменную userInput.

let userInput = prompt();

const sum = 2 + userInput; // вот, что у нас получилось

Здесь использовано ключевое слово const, потому что значение суммы в рамках этого запуска скрипта не изменится

Имя переменной указано в нижнем регистре, потому что мы не хотим, чтобы её значение можно было менять, но это не глобальная переменная.

Один пользователь ввёл значение 2 (number). У нас получилось 4 (number) — значит, всё верно.

Другой пользователь решил узнать, что будет, если ввести “2” (string).

И тут он получает в ответе “22” (string)!

Как это произошло?

JavaScript умеет делать неявное преобразование типов. Это значит, что тип переменной определяется во время её присвоения. Но по ходу программы может меняться.

Если в выражении выполняется сложение числа со строкой, то все числа приводятся к типу строка и происходит конкатенация строк:

2 + "2" + 2 = "222"

Если происходит операция вычитания, деления или умножения, то все значения приводятся к числу:

2 + "2" * 2 = 6

2 + "2" / 2 = 3

2 + "2" - 2 = 20 // здесь приоритет операторов + и - одинаковый, поэтому выражение выполняется слева направо.

Булевый тип данных

Мы изучили, как себя ведут строки при взаимодействии с числами. Теперь рассмотрим, как JavaScript может изменять булевый тип данных — true / false.

Начнём с яблок.

Мы пришли в огород и сорвали с дерева 1 яблоко. У нас появилось одно яблоко.

Если мы зададим JavaScript вопрос: Есть ли у нас яблоко?, то он нам ответит true.

Мы съели яблоко, у нас осталось 0 яблок. Если мы теперь спросим JavaScript Есть ли у нас яблоко?, то он ответит false.

Стоит помнить, что все числа (положительные и отрицательные) при приведении к булевому типу будут выдавать true, а 0 будет выдавать false.

Когда мы проводим какие-либо операции с булевым типом данных, JavaScript может преобразовывать true в 1, а false в 0.

true + 1 = 2;

false - 1 = -1;

true * 2 = 2;
