типы данных в JavaScript

  •

Примитивный (встроенный, базовый) тип — тип данных, предоставляемый языком программирования как базовая встроенная единица языка.

В JavaScript есть 6 примитивных типов данных:

Название типа Пример
number (число) 1, 2, 3.5
string (строка) Обычный текст
boolean (булев, логический тип) true, falsetrue, false
undefined (неопределённый тип) undefined
BigInt (очень большое число) 1234444444444n
Symbol (в ECMAScript 6) new Symbol('a')

number (Число)

Числовой тип данных в JS — это обычные числа:

let one = 1;

let two = 2;

let pi = 3.14;

let negativeOne = -1;

У этого типа данных очень большой интервал возможных значений, но он не бесконечен. Всего для данного типа данных закладывается в памяти 64 бита.

Также существуют специальные значения:

➤+Infinity (положительная бесконечность).

Получим, если превысим максимальное положительное число.

➤-Infinity (отрицательная бесконечность).

Получим, если значение будет меньше минимально возможного числа.

➤NaN (не число)

Получим при разных математических операциях с числами, а также если попробуем строку представить в виде числа.

Мы можем узнать значения самого большого или самого меньшего доступного значения в пределах +/-Infinity. Для этого можно использовать именованные константы, предоставляемые языком: Number.MAX_VALUE или Number.MIN_VALUE.

string (Строка)

В JavaScript строки представляют собой последовательность символов, используемых для представления текста.

В JavaScript есть три способа определить строку в коде:

1.Одиночные кавычки.

2.Двойные кавычки.

3.Обратные кавычки.

let firstString = "Это строка";

let secondString = 'И это строка';

let thirdString = `И даже это строка`;

Между первым и вторым способом разницы нет.

Третий способ называется шаблонные строки — позволяет вставить внутрь строки специальные языковые конструкции ${}. Данные конструкции будут выводить результат работы находящегося внутри них кода:

`Если к 1 прибавить 2 будет ${1 + 2}` // Если к 1 прибавить 2 будет 3

Строки должны начинаться и заканчиваться одинаковыми кавычками.

Если нужно внутрь строки вставить ещё одну кавычку или кавычки, то это нужно делать другим типом кавычек или экранировать с помощью символа обратного слеша — \.

"Строка с 'дополнительными' \" кавычками\" "

// Строка с 'дополнительными' "кавычками"

boolean (Булев, Логический тип)

Может содержать в себе одно из двух значений: true или false.

По простому — это ответ «Да» или «Нет»

let isEquals = true; // да, значения одинаковые

let inputChecked = false; // нет, инпут не выделен

undefined (не определено)

Если вы сразу не указали переменной какое-либо значение, то переменная получит такой тип:

let variable; // undefined

variable = 5; // 5

BigInt (Большое целое)

Тип number может хранить в себе огромный диапазон значений. Если число выходит за этот диапазон, то мы получаем Infinity. В этом случае для переменной подойдёт тип BigInt.

Создавая переменную с типом BigInt мы скажем JavaScript’у, что значение переменной выходит за диапазон типа number. Но оно нам нужно и не надо делать его Infinity.

Чтобы создать значение типа BigInt нужно в конце числа поставить символ n.

const bigInt = 19241924124n

Symbol (Символ)

Символ — это тип данных, который представляет собой уникальный идентификатор.

В отличии от предыдущих типов, Symbol создаётся с помощью специальной функции Symbol().

По спецификации, в качестве ключей для свойств объекта могут использоваться только строки или символы. Ни числа, ни логические значения не подходят. Разрешены только эти два типа данных.

Про объекты подробнее мы поговорим в следующем юните.

Вот как можно создать символ с описанием (именем):

"id": Symbol("id")

Null

Стоит особняком. Также, как и undefined.

Данный тип не относится ни к одному из вышеописанных. Он формирует отдельный тип, в котором имеется только одно значение — null.

Это специальное значение, которое представляет собой «ничего», «пусто» или «значение неизвестно».

let userName = null;

Обычно null используется для присвоения переменной «пустого» или «неизвестного» значения. А undefined — для проверок, была ли переменная назначена.

Оператор typeof

Так как в JavaScript типизация динамическая, возникают ситуации, когда разработчик не уверен, какой именно тип данных хранится в переменной.

Чтобы не допустить ошибок и принять верное решение относительно дальнейших действий, пригодится оператор typeof.

С помощью этого оператора можно моментально узнать о типе данных.

Он возвращает строку с названием типа. Применяется так:

let a = 10;

typeof a; // "number"

a = '10';

typeof a //"string"

let a = 10;

typeof(typeof a); //"string"

С примитивными типами данных оператор работает предсказуемо. Но есть особенности, с которыми мы столкнёмся, когда будем изучать объекты.
